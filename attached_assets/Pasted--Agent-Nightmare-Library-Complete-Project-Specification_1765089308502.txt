Nightmare Library - Complete Project Specification
CRITICAL DEPLOYMENT NOTICE
This project MUST follow Cloudflare Pages deployment rules. DO NOT use Replit-specific configurations, workflows, or conventions. The project will be deployed via GitHub to Cloudflare Pages. Using Replit rules will break the deployment.

Project Overview
Build a private, self-hosted digital library called "Nightmare Library" for Cloudflare Pages deployment via GitHub. This is a single-user application for managing and reading EPUB and PDF books with an obsidian-black themed interface.

Architecture
Polyglot Stack
Performance Layer (C/C++ and Rust → WebAssembly)

C/C++ compiled to WASM using Emscripten
Rust compiled to WASM using wasm-pack
Used for CPU-intensive operations:
EPUB metadata extraction (title, author, cover, page count)
PDF metadata extraction and parsing
Image processing for cover thumbnails
Fast fuzzy string matching for search
WASM modules loaded and executed in Cloudflare Pages Functions
Backend Layer (Cloudflare Pages Functions)

JavaScript or TypeScript files in /functions directory
Runs on Cloudflare Workers runtime (V8 engine)
Handles ALL business logic
MUST provide pre-rendered HTML snippets to frontend
Manages authentication, storage, database operations
Frontend Layer (Pure Vanilla)

HTML, CSS, and vanilla JavaScript ONLY
Located in /frontend directory
NO frameworks (React, Vue, Angular, Svelte)
NO bundlers (Webpack, Vite, Parcel)
NO TypeScript in frontend (only backend)
NO business logic - only renders what backend provides
Receives pre-rendered HTML snippets from backend and inserts them
Pages Structure
ONLY 2 PAGES + READER VIEW
Login Page - index.html (at project root)

Simple password input form
Posts to /functions/auth.js
On success, redirects to dashboard
Dashboard Page - /frontend/dashboard.html

Main library view with all features
Protected by authentication middleware
Contains: header, sidebar, book grid, modals
All UI components rendered from backend snippets
Reader View - /frontend/reader.html

Separate view for reading books (not a "page" in navigation sense)
Renders EPUB using epub.js, PDF using pdf.js
Accessed by clicking a book, not from sidebar navigation
Backend Provides UI to Frontend
The backend MUST return pre-rendered HTML snippets that the frontend inserts directly. This is MANDATORY, not optional.

# Cloudflare Setup Instructions
## 1. Create D1 Database
Run this command to create the D1 database:
\`\`\`bash
npx wrangler d1 create nightmare-library-db
\`\`\`
Copy the `database_id` from the output and paste it into `wrangler.toml` under `[[d1_databases]]`.
## 2. Create KV Namespaces
Run these commands to create KV namespaces:
\`\`\`bash
npx wrangler kv:namespace create "KV_SESSIONS"
npx wrangler kv:namespace create "KV_CACHE"
\`\`\`
Copy each `id` from the output and paste into `wrangler.toml` under the corresponding `[[kv_namespaces]]`.
## 3. Run Database Migrations
After creating D1, run the SQL schema:
\`\`\`bash
npx wrangler d1 execute nightmare-library-db --remote --file=./migrations/0001_init.sql
\`\`\`
## 4. Set All Required Secrets
### SECRET: PASSWORD
- **Purpose**: Master password for login authentication
- **Required Data**: A strong password string (min 12 characters recommended)
- **Command**: `npx wrangler secret put PASSWORD`
### SECRET: JWT_SECRET
- **Purpose**: Signs authentication tokens
- **Required Data**: Random 64+ character string (use a password generator)
- **Command**: `npx wrangler secret put JWT_SECRET`
### SECRET: STORAGE_PROVIDER_1_TYPE
- **Purpose**: Type of primary storage provider
- **Required Data**: One of: `r2`, `s3`, `gcs`, `backblaze`
- **Command**: `npx wrangler secret put STORAGE_PROVIDER_1_TYPE`
### SECRET: STORAGE_PROVIDER_1_BUCKET
- **Purpose**: Bucket name for primary storage
- **Required Data**: Your bucket name (e.g., `nightmare-library-books`)
- **Command**: `npx wrangler secret put STORAGE_PROVIDER_1_BUCKET`
### SECRET: STORAGE_PROVIDER_1_ACCESS_KEY
- **Purpose**: Access key/ID for storage provider
- **Required Data**: 
  - For R2: R2 Access Key ID from Cloudflare dashboard
  - For S3: AWS Access Key ID
  - For GCS: Service account client email
- **Command**: `npx wrangler secret put STORAGE_PROVIDER_1_ACCESS_KEY`
### SECRET: STORAGE_PROVIDER_1_SECRET_KEY
- **Purpose**: Secret key for storage provider
- **Required Data**:
  - For R2: R2 Secret Access Key from Cloudflare dashboard
  - For S3: AWS Secret Access Key
  - For GCS: Service account private key (JSON string)
- **Command**: `npx wrangler secret put STORAGE_PROVIDER_1_SECRET_KEY`
### SECRET: STORAGE_PROVIDER_1_ENDPOINT (Optional for S3/R2)
- **Purpose**: Custom endpoint URL
- **Required Data**: 
  - For R2: `https://<account_id>.r2.cloudflarestorage.com`
  - For S3: Leave empty or custom endpoint
- **Command**: `npx wrangler secret put STORAGE_PROVIDER_1_ENDPOINT`
### SECRET: STORAGE_PROVIDER_1_REGION (Optional)
- **Purpose**: Storage region
- **Required Data**: Region code (e.g., `us-east-1`, `auto` for R2)
- **Command**: `npx wrangler secret put STORAGE_PROVIDER_1_REGION`
## 5. Connect GitHub to Cloudflare Pages
1. Go to Cloudflare Dashboard → Pages
2. Click "Create a project" → "Connect to Git"
3. Authorize GitHub and select your repository
4. Configure build settings:
   - Project name: `nightmare-library`
   - Production branch: `main`
   - Build command: `npm run build` (or leave empty if no build)
   - Build output directory: `/`
5. Go to Settings → Functions → Add bindings for D1 and KV
6. Go to Settings → Environment Variables → Add all secrets
7. Deploy!

Database Schema (D1)
migrations/0001_init.sql
-- Books table
CREATE TABLE IF NOT EXISTS books (
    id TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    author TEXT,
    storage_provider TEXT NOT NULL,
    storage_id TEXT NOT NULL,
    cover_url TEXT,
    file_type TEXT NOT NULL,
    file_size INTEGER,
    tags TEXT,
    total_pages INTEGER,
    uploaded_at INTEGER NOT NULL,
    last_read_at INTEGER,
    last_read_position INTEGER DEFAULT 0
);
-- Shelves table
CREATE TABLE IF NOT EXISTS shelves (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    color TEXT DEFAULT '#bb86fc',
    position INTEGER DEFAULT 0,
    created_at INTEGER NOT NULL
);
-- Shelf items (many-to-many relationship)
CREATE TABLE IF NOT EXISTS shelf_items (
    shelf_id TEXT NOT NULL,
    book_id TEXT NOT NULL,
    added_at INTEGER NOT NULL,
    PRIMARY KEY (shelf_id, book_id),
    FOREIGN KEY (shelf_id) REFERENCES shelves(id) ON DELETE CASCADE,
    FOREIGN KEY (book_id) REFERENCES books(id) ON DELETE CASCADE
);
-- Reading progress
CREATE TABLE IF NOT EXISTS progress (
    book_id TEXT PRIMARY KEY,
    percent INTEGER NOT NULL DEFAULT 0,
    current_page INTEGER,
    current_chapter TEXT,
    last_read_at INTEGER NOT NULL,
    FOREIGN KEY (book_id) REFERENCES books(id) ON DELETE CASCADE
);
-- Indexes
CREATE INDEX IF NOT EXISTS idx_books_uploaded ON books(uploaded_at DESC);
CREATE INDEX IF NOT EXISTS idx_books_last_read ON books(last_read_at DESC);
CREATE INDEX IF NOT EXISTS idx_progress_last_read ON progress(last_read_at DESC);
Authentication System
Flow
User visits / (index.html) - login page
User enters password, form POSTs to /functions/auth
Backend validates password against PASSWORD secret
If valid:
Generate JWT token signed with JWT_SECRET
Store session in KV_SESSIONS with 24-hour TTL
Set cookie: NMLR_SESSION={token}; HttpOnly; Secure; SameSite=Strict; Path=/; Max-Age=86400
Return redirect to /frontend/dashboard.html
If invalid: Return error message
Middleware Protection
The _middleware.ts file MUST:

Check for NMLR_SESSION cookie on all requests to /frontend/* and /api/*
Validate token against KV_SESSIONS
Redirect to / if invalid or missing
Allow access to / (login page) without auth
Cookie Requirements
HttpOnly - JavaScript cannot access
Secure - HTTPS only
SameSite=Strict - Prevents CSRF
Path=/ - Available on all routes
Max-Age=86400 - 24 hours

## 4. Set All Required Secrets
### SECRET: PASSWORD
- **Purpose**: Master password for login authentication
- **Required Data**: A strong password string (min 12 characters recommended)
- **Command**: `npx wrangler secret put PASSWORD`
### SECRET: JWT_SECRET
- **Purpose**: Signs authentication tokens
- **Required Data**: Random 64+ character string (use a password generator)
- **Command**: `npx wrangler secret put JWT_SECRET`
### SECRET: STORAGE_PROVIDER_1_TYPE
- **Purpose**: Type of primary storage provider
- **Required Data**: One of: `r2`, `s3`, `gcs`, `backblaze`
- **Command**: `npx wrangler secret put STORAGE_PROVIDER_1_TYPE`
### SECRET: STORAGE_PROVIDER_1_BUCKET
- **Purpose**: Bucket name for primary storage
- **Required Data**: Your bucket name (e.g., `nightmare-library-books`)
- **Command**: `npx wrangler secret put STORAGE_PROVIDER_1_BUCKET`
### SECRET: STORAGE_PROVIDER_1_ACCESS_KEY
- **Purpose**: Access key/ID for storage provider
- **Required Data**: 
  - For R2: R2 Access Key ID from Cloudflare dashboard
  - For S3: AWS Access Key ID
  - For GCS: Service account client email
- **Command**: `npx wrangler secret put STORAGE_PROVIDER_1_ACCESS_KEY`
### SECRET: STORAGE_PROVIDER_1_SECRET_KEY
- **Purpose**: Secret key for storage provider
- **Required Data**:
  - For R2: R2 Secret Access Key from Cloudflare dashboard
  - For S3: AWS Secret Access Key
  - For GCS: Service account private key (JSON string)
- **Command**: `npx wrangler secret put STORAGE_PROVIDER_1_SECRET_KEY`
### SECRET: STORAGE_PROVIDER_1_ENDPOINT (Optional for S3/R2)
- **Purpose**: Custom endpoint URL
- **Required Data**: 
  - For R2: `https://<account_id>.r2.cloudflarestorage.com`
  - For S3: Leave empty or custom endpoint
- **Command**: `npx wrangler secret put STORAGE_PROVIDER_1_ENDPOINT`
### SECRET: STORAGE_PROVIDER_1_REGION (Optional)
- **Purpose**: Storage region
- **Required Data**: Region code (e.g., `us-east-1`, `auto` for R2)
- **Command**: `npx wrangler secret put STORAGE_PROVIDER_1_REGION`

Database Schema (D1)
migrations/0001_init.sql
-- Books table
CREATE TABLE IF NOT EXISTS books (
    id TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    author TEXT,
    storage_provider TEXT NOT NULL,
    storage_id TEXT NOT NULL,
    cover_url TEXT,
    file_type TEXT NOT NULL,
    file_size INTEGER,
    tags TEXT,
    total_pages INTEGER,
    uploaded_at INTEGER NOT NULL,
    last_read_at INTEGER,
    last_read_position INTEGER DEFAULT 0
);
-- Shelves table
CREATE TABLE IF NOT EXISTS shelves (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    color TEXT DEFAULT '#bb86fc',
    position INTEGER DEFAULT 0,
    created_at INTEGER NOT NULL
);
-- Shelf items (many-to-many relationship)
CREATE TABLE IF NOT EXISTS shelf_items (
    shelf_id TEXT NOT NULL,
    book_id TEXT NOT NULL,
    added_at INTEGER NOT NULL,
    PRIMARY KEY (shelf_id, book_id),
    FOREIGN KEY (shelf_id) REFERENCES shelves(id) ON DELETE CASCADE,
    FOREIGN KEY (book_id) REFERENCES books(id) ON DELETE CASCADE
);
-- Reading progress
CREATE TABLE IF NOT EXISTS progress (
    book_id TEXT PRIMARY KEY,
    percent INTEGER NOT NULL DEFAULT 0,
    current_page INTEGER,
    current_chapter TEXT,
    last_read_at INTEGER NOT NULL,
    FOREIGN KEY (book_id) REFERENCES books(id) ON DELETE CASCADE
);
-- Indexes
CREATE INDEX IF NOT EXISTS idx_books_uploaded ON books(uploaded_at DESC);
CREATE INDEX IF NOT EXISTS idx_books_last_read ON books(last_read_at DESC);
CREATE INDEX IF NOT EXISTS idx_progress_last_read ON progress(last_read_at DESC);
Authentication System
Flow
User visits / (index.html) - login page
User enters password, form POSTs to /functions/auth
Backend validates password against PASSWORD secret
If valid:
Generate JWT token signed with JWT_SECRET
Store session in KV_SESSIONS with 24-hour TTL
Set cookie: NMLR_SESSION={token}; HttpOnly; Secure; SameSite=Strict; Path=/; Max-Age=86400
Return redirect to /frontend/dashboard.html
If invalid: Return error message
Middleware Protection
The _middleware.ts file MUST:

Check for NMLR_SESSION cookie on all requests to /frontend/* and /api/*
Validate token against KV_SESSIONS
Redirect to / if invalid or missing
Allow access to / (login page) without auth
